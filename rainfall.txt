Propriete systeme:

GCC stack protector support: Enabled	Le compilateur supporte la protection contre les débordements de pile (stack canaries), mais ce n’est pas forcément activé pour ton binaire.
Strict user copy checks: Disabled	Moins important ici, c’est une protection contre des bugs en copie de mémoire noyau → utilisateur.
Restrict /dev/mem access: Enabled	Accès limité à la mémoire physique via /dev/mem (sécurité renforcée).
Restrict /dev/kmem access: Enabled	Pareil pour /dev/kmem (vieilles interfaces mémoire noyau).
grsecurity / PaX: No GRKERNSEC	Aucune protection PaX/Grsecurity active sur le noyau.
Kernel Heap Hardening: No KERNHEAP	Pas de protection spéciale sur le tas du noyau.
ASLR (Address Space Layout Randomization): Off	La mémoire n’est pas randomisée, donc les adresses sont toujours les mêmes → très utile !



level0 :

Binary checksec:
RELRO: No RELRO	Aucune protection RELRO → tu peux réécrire certaines parties de la table GOT.

NX: NX enabled	La mémoire non exécutable est activée → tu ne peux pas exécuter du shellcode sur la stack. Tu devras faire un ret2libc plus tard.
PIE: No PIE	Le binaire n’est pas position-independant → les adresses dans le binaire sont toujours les mêmes. C’est très bien pour toi !
